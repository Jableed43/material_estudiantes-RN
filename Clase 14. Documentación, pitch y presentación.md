## 1. El README Profesional {#readme-intro}

Un buen código sin documentación es un código que nadie usará (o querrá pagar por él).
- **README.md**: La puerta de entrada a tu proyecto en GitHub. Debe explicar qué hace, cómo se instala y qué usaste.
- **Screenshots**: Incluye 2 o 3 imágenes de las mejores pantallas de tu app para venderla en segundos.

---

## 2. El Pitch: ¿Cómo presentar tu idea? {#que-es-pitch}

Tienes **3 minutos** para impresionar a un cliente o un reclutador.
1. **El Problema**: ¿Qué dolor o necesidad resuelve tu app?
2. **La Solución**: Cómo tu app hace la vida más fácil.
3. **La Demo**: Muestra la app funcionando en vivo (es lo más importante).
4. **Cierre**: Un resumen rápido y agradecer al público.

---

## 3. Preparación Técnica (El Demo Day) {#presentacion}

Evita el **"Efecto Demo"** (que algo falle justo cuando lo muestras):
- **Plan B**: Graba un video de 1 minuto usando la app por si el internet o el servidor fallan.
- **Datos de Prueba**: Ten un "Usuario de Prueba" con tareas o productos ya cargados para no mostrar una app vacía.

---

## 4. Requisitos de Entrega {#checklist-final}

Para dar por finalizado el curso, asegúrate de cumplir con:
- [ ] Repositorio en GitHub público o compartido.
- [ ] Archivo README con la documentación completa.
- [ ] APK de Android o link de Expo para probar la aplicación.
- [ ] Al menos 3 funcionalidades principales funcionando (CRUD completo).
- [ ] Los estilos y el diseño son consistentes (Colores, Logos, etc).

---

## 5. ¡Demo Day! {#demo-day}

Presentarás tu proyecto final frente a tus compañeros:
1. Conecta tu celular o emulador a la pantalla compartida.
2. Realiza tu pitch de máximo 3 minutos.
3. Responde preguntas técnicas: "¿Cómo hiciste la navegación?" o "¿Dónde guardas los datos?".
4. ¡Disfruta el cierre de todo tu esfuerzo!
